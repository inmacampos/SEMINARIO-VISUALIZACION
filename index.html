RESUMEN

La base de datos utilizada en este trabajo es un histórico de un e-commerce que vende neumáticos por internet. La base de datos tiene 5000 elementos acumulados entre el año 2016 y el 2019. Como elementos la base de datos tiene el número de numáticos,la marca y el segmento de la marca, la facturación, la fecha e información acerca de la localización (Población, Provincia y CP).

Me gustaría hace un análisis del funcionamiento de la e-commerce.
Para ello se mostrará cual es el tipo de pedido más común,sindo 2 y 4 la catidad de neumáticos más común.

El segmento de neumáticos que produce una mayor facturación es el segmento denominado calidad-precio y produce un 22% en pedido de dos ruedas y un 19 % en pedidos con cuatro ruedas.

La marca que más facturación produce a la e-commerce el la marca Hifly dentro del segmento de los nuemáticos ecómicos con la mitad de la facturación de este segmento. Dentro de los numáticos Premium las Michelin son las que más facturación producen.

El mapa que representa las ventas por localidad muestra que las zonas de costa son las que más ventas producen, después de Madrid y Barcelona. Curiosamente en Coruña hay un municipio con ventas significativamente mayores a lo esperado.

Abril junto con octubre y noviembre son los meses que más nuemáticos venden, preparando el coche para las vacaciones de verano y Navidad, y agosto junto con diciembre los meses donde se producen menos ventas.

Este trabajo tiene como audencia los compañeros de clase de la asignatura de visualización.

Número de pedidos más común

Este gráfico representa el número de neumáticos vendidos en cada pedido. En este caso la variable cantidad se ha utilizado como variable cualitativa y la suma de la cantidad como variable cuantitativa. Son dos usos diferentes de una misma variable. Este gráfico se podría haber representado mediante un gráfico circular, pero debido a la gran diferencia entre 2 y 4 con pedido con 3 y 5 neumáticos, esto últimos no se observan en el gráfico circular.

Este gráfico se ha realizado con python usando la librería seaborn.


Porcentaje factuación por segmento y cantidad de neumáticos

En este gráfico se introducen dos variables más, la facturación y el segmento. Se ha utilizado un gráfico de barras para facilitar la comparación entre los diferentes segmentos. La alternativa de un gráfico de barras acumuladas, no permitía comparar entre los diferentes segmentos fácilmente.


Este gráfico se ha realizado con python usando la librería seaborn.


Facturación según marca y segmento

Mediante un mapa de árbol se cruzan dos variable cuantitativas (segmento y marca) con una variable cuantitativa como facturación. Se ha elegido este gráfico para tener una comparación visual y aproximada de lo que factura cada segmento y cada marca. Para una comparación más al detelle se debería de haber elegido un gráfico de barras.


Este gráfico se ha realizado con tableau online.


Meses con más pedidos

Se ha utilizado un gráfico de areas para mostrar la tendencia del número de pedidos a lo largo de dos años.
